# DashType v1.2

DashType v1.2 adds dynamic snippet commands, more reliable cursor placement, and a smoother accessibility setup flow.

## What's New

- New `Commands` panel in the snippet editor for inserting supported snippet commands
- New `{cursor}` command to place the caret at an exact location after snippet expansion
- New `{clipboard}` command to insert the current clipboard contents into a snippet
- New `{addtext}` command to accept custom inline text with the `{Custom Text}/trigger` format

## Improvements

- Improved cursor placement reliability after expansion with a more direct caret-positioning approach
- Added safeguards so only one `{cursor}` and one `{addtext}` command can be used in a snippet
- Updated command cards to show inline token hints and clearer usage guidance
- Dashboard now opens a bit wider by default to better fit the new commands panel
- Accessibility permission now refreshes automatically after approval, without requiring an extra click back into the app

## Examples

- `Hello {cursor}, how are you?`
- `Current clipboard: {clipboard}`
- Snippet content: `Hi {addtext}, how are you?`
- Typed trigger: `{John}/greet`
- Result: `Hi John, how are you?`

## Notes

- Accessibility permission is still required for global snippet expansion
- The release asset for this version is `DashType_v1.2.dmg`
